Collaborative Testing Approach

At Exago Inc



Manual testing is the process of manually testing software for defects. It requires a tester to play the role of an end user whereby they use most of the application's features to ensure correct behavior.

Manually executing test cases is multiple days' worth of work; my job was to find a way to streamline this process to allow for the quality assurance to take a more analytical approach to testing.



Key Learnings

  • Working with a complex projects under tight timelines
  • Test Case development
  • Test Case tracking systems
  • Developing education resources
  • Effective communication across multiple departments

Problem Statement


Too many test cases not enough testers. This is the issue we were facing in the quality assurance department when the company decided to overhaul three of our designers and targerted it to the same major release. This did not allow us enough time with the development branch to test due to tight timelines.

Proposed Solution


The solution I proposed was to get the entire company involved in testing.

The immediate issues we faced Not everyone in the company was a technical user and had the capability of setting up a robust testing environment.
Wants and Needs:

  • Need to build an enviroment where non technical users could just access the application and not install it locally
  • Needs to educate users on the new product feature for them to be able to test it
  • Want to get as much value out of the testers as possible

Frustrations:

  • Having to put together these resources and making them easily accessible
  • Non technical users struggling with the high learning curve
  • Getting users up to speed takes a long time

Results


More people testing = more bugs found (shocking right)
